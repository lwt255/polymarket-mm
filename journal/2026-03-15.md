## Session Log

### Session 1: Split Straddle Autoresearch — Build, Test, Run

#### What We Built

A second autoresearch loop for the split straddle strategy, running in parallel with the existing arb autoresearch. The split straddle is a fundamentally different approach:

- **Arb**: Place limit BUY orders on both sides, hope both fill at < $1.00 combined
- **Split Straddle**: Split USDC into UP+DOWN tokens, predict the loser via Chainlink direction, sell the loser at bid price

**New files created** (6 files in `src/scripts/autoresearch/`):

| File | Purpose |
|------|---------|
| `split-types.ts` | 12 tunable parameters (checkpointSeconds, minClMove, maxLoserBid, adaptive timing, etc.) |
| `split-params.ts` | AI-editable params file |
| `split-bot.ts` | Dry-run bot — monitors live markets, predicts loser, simulates sell |
| `split-evaluate.ts` | Scoring: accuracy-weighted composite |
| `split-loop.ts` | Orchestrator with `--no-git` mode for parallel runs |
| `split-straddle-playbook.md` | AI research playbook |

**Key design decision — `--no-git` mode**: Since the arb autoresearch was already running on a git branch, we couldn't create another branch simultaneously. Added `--no-git` flag that does in-memory param revert (saves previous file content, restores on rejection) instead of git commit/revert. Same logic, no git conflicts.

#### What the AI Discovered

1. **Adaptive checkpoints are essential** — Iter 1 turned them on and they stayed on for all accepted runs. They detect direction flips and skip those markets.
2. **$2 minimum CL move is the sweet spot** — $10 was too restrictive (few trades), $0.50-$1 was too noisy (wrong predictions). $2 filters coin flips without missing real moves.
3. **40¢ max loser bid is better than 30¢** — captures higher-profit markets. But 50¢ is too high (iter 12 failed at 50% accuracy).
4. **T-20s checkpoint slightly better than T-30s** — iter 11 (T-20s) was the best accepted run.
5. **Depth ratio matters** — dropping from 0.3 to 0.1 led to losses (iters 9, 14). Thin books = unreliable.

#### The Core Finding (Confirmed)

**83.7% accuracy on Saturday is not profitable.** The asymmetry kills it:
- Correct prediction: win loser bid (typically 5-25¢ per dollar)
- Wrong prediction: lose (100¢ - loser bid) = 75-95¢ per dollar
- Need ~90%+ accuracy to break even, ~93%+ to be meaningfully profitable

Historical backtests showed 90-97% accuracy during weekday peak hours (10 AM - 3 PM ET). **Monday is the real test.**

### Arb Autoresearch Run 2 (still running, 14/20 iterations)

The arb strategy continues to struggle. Only 2 accepted runs (iters 2 and 10) out of 14. Single-fill losses consistently dominate — the AI can't find parameter combinations that prevent the occasional catastrophic single fill.

**Key arb findings so far**:
- offset=0.03 with shares=10 is the only config that ever scored positive
- minSpreadCents=3 filtered out ALL markets on Saturday (spreads too tight)
- fillThresholdCents=1 didn't help
- entryDelaySeconds=30 didn't help
- The AI is stuck in a loop trying to reproduce iter 2's result

**Conclusion**: The arb approach has a structural problem on these markets. Both-fill profit is thin (~2-6¢) and single-fill losses are massive (~50-90¢). The split straddle is the stronger strategy.
