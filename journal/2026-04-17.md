## 📝 What Happened (chronological)

### Start: "Is the strategy broken?"
Live bot ran Apr 9-17 with mixed results. After clean period (Apr 15-17 post-taker-first-fix) showed −$0.48/tr, user asked if sim/strategy still has edge.

### First analysis — sim said YES
Ran v4-sim over 28 days collector data → +$0.77/tr at 69% WR. Consistent across 3/7/14/28 day windows. "Edge is real, not decaying."

### Cross-reference revealed 52-trade gap
Compared sim qualifying set (339 trades Apr 15-17) vs live ledger (134 filled):
- 93 overlap
- 52 bot traded / sim rejected — these LOST $134 at 48% WR
- 210 sim said take / bot missed

### Drift check exposed timing issue
For 124 live trades, compared bot's recorded ledger values vs collector's T-30 snap for same slug:
- **73/124 (59%)** had ask gap >2¢
- **9 (7%)** had different leader side entirely
- Extreme examples: 14¢ gap, 17¢ gap, 25¢ gap
- BTC example: bot thought UP leader at 56¢, collector saw DOWN leader at 51¢ — opposite direction

### Ruled out several hypotheses
- **Lookahead in collector data?** Checked — all collector "T-30" snaps were at sec=26-30, not late. Fine.
- **Stale cache?** Checked quoteAgeMs — all <500ms, fresh. Fine.
- **Silent fetch failures?** Instrumented bot with snap-diagnostics, ran dry-run for 40 entries. ZERO fetch failures.

### Dry-run diagnostics confirmed timing is real
Bot's actual T-33 snaps vs collector's T-30 snaps on SAME market, same 3-second window:
- 40% differ by >2¢
- Average ask gap: 3.3¢
- 2 side mismatches out of 40

**This is 3 seconds of normal market movement, not a bug.**

### Re-ran sim with honest timing
Forced sim to only use snapshots at T-33 or later (never peek ahead). Edge drops:
- T-30 (cheating): +$0.50/tr, +$906 total
- **T-33 (honest): +$0.27/tr, +$523 total**
- T-40 (conservative): +$0.15/tr, +$306 total

Half the original edge was lookahead. Real edge is smaller but positive.

### Per-trade match analysis
For 79 trades where bot and sim AGREED to trade:

- "Friction": −$0.36/tr on these trades

For 55 trades bot took but sim rejected: only −$11 total expected PnL (not the $134 we first thought — that was at cheating T-30 sim).

### The "263 missed" analysis
Sim qualifying set had 342 trades Apr 15-17. Bot captured 79. Sim predicted the 263 missed were 71% WR, +$189 expected PnL. If bot had caught them all, total would be +$163 vs −$78 actual.

User's interpretation: "bot selects the worst slice of qualifying trades." Probable causes identified (but not individually quantified):

2. Sequential entry-window processing (4-8 markets, 1-2s each) = late markets get T-20 data
3. Taker max-price cap ($0.75) blocks trades where ask drifts up
4. Mutex hits when multiple cryptos end same timestamp

User said halts are fine (prevent blowups). Focus on #2 and #3.
