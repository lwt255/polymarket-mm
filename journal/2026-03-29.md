## The Redeem Mystery — SOLVED

### What we thought
- Redeems weren't working, winning tokens disappeared, no USDC returned

### What actually happened
1. **Trades DO settle on-chain** — confirmed via ERC1155 TransferSingle events from CTF Exchange to our EOA
2. **Redeems DO work** — tokens get burned, USDC returns to wallet
3. **The math proves it**: $133.79 - $66.27 (trade costs) + $23 (2 wins) - ~$10 (gas) = ~$80.52 expected vs $79.14 actual — **within $1.38**
4. **The bot just didn't KNOW it won** — Polymarket API gave wrong resolution when polled too early, so bot logged wins as losses
5. **The $79.14 balance already includes the winning payouts**

### Why the bot logs wrong resolutions
- Bot polls Polymarket API ~15 seconds after candle ends
- The API sometimes returns stale/pre-resolution prices at that point
- The DEFINITIVE resolution comes later (30-90 seconds)
- Bot records the early (wrong) answer, but the on-chain redeem uses the correct resolution

### Resolution is NOT the bot's job
- The bot buys shares. Polymarket resolves them. The CTF contract pays out.
- What the bot THINKS the resolution is doesn't affect the actual payout
- The wallet balance is the truth — not the bot's log

## On-Chain Research Findings

### Settlement Flow (confirmed via contract source + on-chain events)
1. CLOB matches order off-chain
2. Operator calls `matchOrders()` on CTF Exchange — settles on-chain
3. Tokens transfer to EOA via ERC1155 TransferSingle
4. `redeemPositions()` burns tokens and returns USDC to caller
5. If condition not yet resolved on-chain, redeem REVERTS (tokens safe)
